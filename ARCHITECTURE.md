## 🧩 核心组件

### 1. 测试执行器 (api_perf_test.py)

**职责**：执行 evalscope perf 测试并收集结果

**核心类**：
- `StabilityTest`: 主测试控制器
  - 构建测试命令
  - 执行测试循环
  - 管理测试状态
  
- `PerfOutputParser`: 输出解析器
  - 正则表达式匹配性能指标
  - 提取表格数据
  
- `TestResult`: 测试结果容器
  - 存储单次测试数据
  - 序列化为 JSON

**数据流**：
```
配置 → 命令构建 → subprocess 执行 → 输出捕获 → 解析 → 存储
```

### 2. 报告生成器 (generate_report.py)

**职责**：将测试结果转换为可视化 HTML 报告

**核心类**：
- `HTMLReportGenerator`: 报告生成器
  - 加载 JSON 数据
  - 提取时间序列
  - 计算统计指标
  - 生成 HTML

**数据处理流程**：
```
JSON → 时间序列提取 → 统计计算 → 图表数据准备 → HTML 渲染
```

### 3. 配置管理

**配置文件结构**：
```json
{
  "api": "openai",           // API 类型
  "url": "...",              // API URL
  "api_key": "...",          // API Key
  "model": "...",            // 模型名称
  "parallel": 8,             // 并发数
  "number": 50,              // 请求数
  "test_schedule": {         // 测试计划
    "total_tests": 48,       // 总次数
    "interval_seconds": 1800 // 间隔
  }
}
```

## 🔧 技术实现细节

### 1. 输出解析

使用正则表达式解析 evalscope perf 的表格输出：

```python
METRICS_PATTERN = re.compile(
    r'│\s*(\d+)\s*│\s*(\S+)\s*│\s*([\d.]+)\s*│\s*'
    r'([\d.]+)\s*│\s*([\d.]+)\s*│\s*'
    r'([\d.]+)\s*│\s*([\d.]+)\s*│\s*'
    r'([\d.]+)\s*│\s*([\d.]+)\s*│\s*'
    r'([\d.]+)\s*│\s*([\d.]+)%\s*│'
)
```

匹配的指标：
1. Concurrency (并发数)
2. Rate (速率)
3. RPS (每秒请求数)
4. Avg Latency (平均延迟)
5. P99 Latency (P99 延迟)
6. Avg TTFT (平均首字延迟)
7. P99 TTFT (P99 首字延迟)
8. Avg TPOT (平均每 token 时间)
9. P99 TPOT (P99 每 token 时间)
10. Gen. toks/s (生成速度)
11. Success Rate (成功率)

### 3. 统计计算

计算每个指标的最小值、最大值、平均值：

```python
def _calculate_statistics(self, time_series):
    stats = {}
    for key in metrics_keys:
        values = [v for v in time_series[key] if v > 0]
        stats[key] = {
            'min': min(values),
            'max': max(values),
            'avg': sum(values) / len(values)
        }
```

## 🎯 设计原则

1. **模块化**: 每个组件职责单一，易于维护
2. **可配置**: 通过配置文件灵活调整参数
3. **可扩展**: 易于添加新的指标和图表
4. **容错性**: 处理测试失败，保存中间结果
5. **可读性**: 清晰的代码结构和注释

## 🔄 扩展点

### 1. 添加新指标

### 2. 自定义图表

### 3. 其他测试工具

### 4. 数据导出

添加 CSV、Excel 等格式的导出功能。

## 📈 性能考虑

1. **内存**: 使用流式处理，避免一次性加载大量日志
2. **并发**: 测试间隔使用 `time.sleep()`，不占用 CPU
3. **存储**: JSON 文件使用增量保存，避免数据丢失
4. **渲染**: HTML 使用 CDN 加载 Chart.js，减少本地资源

## 🔒 安全考虑

1. **API Key**: 不记录到日志文件，仅存储在 JSON 结果中
2. **配置文件**: 提供 `.example` 模板，避免提交敏感信息
3. **输入验证**: 检查配置参数的有效性
